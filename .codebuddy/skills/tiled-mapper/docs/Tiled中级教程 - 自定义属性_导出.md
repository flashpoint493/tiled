## 摘要
本期视频讲解了 Tiled 地图编辑器中的“自定义属性”功能。通过创建项目工程、使用自定义类型编辑器，我们可以把角色、敌人、物品等对象常用属性预先定义成标准模板，避免手动重复添加属性时出现遗漏、拼写错误或数据结构混乱的问题，从而提升关卡配置效率和项目规范性。


### 亮点
- Tiled 的自定义属性可以理解为对象属性模板，适合为玩家、敌人、物品等游戏对象建立统一的数据结构。
- 使用自定义属性前，需要先创建 Tiled 项目工程，否则“自定义类型编辑器”无法启用。
- 通过创建 `actor` 类，可以把角色常用的属性，如 `animation`、`gravity`、`health`、`sound`、`tag` 等集中管理。
- 自定义属性能够减少手动输入属性名的次数，从源头避免属性遗漏和拼写错误。
- 对于固定范围的属性，例如 `tag`，可以使用枚举类型限制可选值，如 `player`、`enemy`、`item`。
- 类类型可以相互嵌套，例如将 `sound` 单独定义为一个类，再嵌入到 `actor` 类中，用来规范跳跃、受伤、死亡等音效路径。
- Tiled 项目文件本质上是 JSON 文件，其中保存了面向整个项目的属性类型配置。
- 建议以后每次使用 Tiled 制作地图时都先创建项目工程，因为它不仅支持自定义属性，也会在后续工作流中发挥更多作用。

### 思考 
1. **[为什么使用自定义属性前必须先创建 Tiled 项目？](https://bibigpt.co/search?q=%E4%B8%BA%E4%BB%80%E4%B9%88%E4%BD%BF%E7%94%A8%E8%87%AA%E5%AE%9A%E4%B9%89%E5%B1%9E%E6%80%A7%E5%89%8D%E5%BF%85%E9%A1%BB%E5%85%88%E5%88%9B%E5%BB%BA%20Tiled%20%E9%A1%B9%E7%9B%AE%EF%BC%9F)**
  - 因为自定义属性类型是保存在项目级配置中的，只有创建项目后，Tiled 才能启用“自定义类型编辑器”，并让这些属性模板在整个项目的地图中复用。
- 2. **[自定义属性相比手动添加属性有什么优势？](https://bibigpt.co/search?q=%E8%87%AA%E5%AE%9A%E4%B9%89%E5%B1%9E%E6%80%A7%E7%9B%B8%E6%AF%94%E6%89%8B%E5%8A%A8%E6%B7%BB%E5%8A%A0%E5%B1%9E%E6%80%A7%E6%9C%89%E4%BB%80%E4%B9%88%E4%BC%98%E5%8A%BF%EF%BC%9F)**
  - 它可以提前定义好属性名称、类型和默认结构，使用时直接套用模板，避免重复劳动，也能防止漏填属性或拼错属性名。
- 3. **[什么时候应该使用枚举类型？](https://bibigpt.co/search?q=%E4%BB%80%E4%B9%88%E6%97%B6%E5%80%99%E5%BA%94%E8%AF%A5%E4%BD%BF%E7%94%A8%E6%9E%9A%E4%B8%BE%E7%B1%BB%E5%9E%8B%EF%BC%9F)**
  - 当某个属性只能从固定选项中选择时，例如角色标签 `tag` 只能是 `player`、`enemy`、`item`，就适合使用枚举类型来限制输入范围。


### 术语解释
- **自定义属性**: Tiled 中用于为对象、图块或地图元素预设属性结构的功能，可以把常用配置做成模板并重复使用。
- **自定义类型编辑器**: Tiled 中用于创建和管理自定义类、枚举等属性类型的工具，需要在创建项目后才能使用。
- **类（Class）**: 一组属性的集合，可以用来描述某类对象的标准数据结构，例如 `actor` 类可以包含生命值、动画、音效、标签等属性。
- **枚举（Enum）**: 一种限制取值范围的数据类型，适合用于固定选项，例如角色类型只能选择 `player`、`enemy` 或 `item`。
- **JSON**: 一种常见的数据存储格式，Tiled 的项目文件和部分复杂属性配置都可以用 JSON 形式保存和读取。
