## 摘要
本期教程讲解 Tiled 的 **World（世界）功能**，用于把多个独立的小地图文件组织成一个可视化的无缝大世界。通过 World 文件，我们既能避免单张超大地图带来的性能和管理问题，又能在编辑时看到相邻地图的连接关系，从而更方便地绘制地图边缘过渡。


### 亮点
- Tiled 的 World 功能不是一种新的地图格式，而是一个用于管理多个地图文件空间位置的配置容器。
- 当地图被加入 World 后，打开其中一张地图时，Tiled 可以自动显示周围相邻地图，帮助开发者进行跨地图编辑。
- 使用 World 工具可以把当前地图添加到世界中，也可以继续添加其他地图并通过拖拽调整它们的位置。
- 多张小地图在 World 视图中可以像一张大地图一样直观排列，方便检查道路、地形和边缘过渡是否自然衔接。
- World 文件本质上记录了一个地图数组，每个子地图都有对应的文件路径和坐标位置。
- 对大多数独立游戏来说，手动添加和摆放地图已经足够；如果地图数量达到成百上千张，可以使用 Pattern Match 自动匹配和摆放地图。
- World 功能能在保持小地图文件轻量、高性能的同时，为开发者提供接近开放世界大地图的全局编辑体验。

### 思考 
1. **[Tiled 的 World 功能适合什么场景使用？](https://bibigpt.co/search?q=Tiled%20%E7%9A%84%20World%20%E5%8A%9F%E8%83%BD%E9%80%82%E5%90%88%E4%BB%80%E4%B9%88%E5%9C%BA%E6%99%AF%E4%BD%BF%E7%94%A8%EF%BC%9F)**
  - 它适合制作开放世界、连续关卡或由多个小地图拼接而成的大地图项目，尤其适合需要处理地图边缘衔接和全局布局的游戏。
- 2. **[World 文件会把所有地图合并成一张大地图吗？](https://bibigpt.co/search?q=World%20%E6%96%87%E4%BB%B6%E4%BC%9A%E6%8A%8A%E6%89%80%E6%9C%89%E5%9C%B0%E5%9B%BE%E5%90%88%E5%B9%B6%E6%88%90%E4%B8%80%E5%BC%A0%E5%A4%A7%E5%9C%B0%E5%9B%BE%E5%90%97%EF%BC%9F)**
  - 不会。World 文件只是一个配置容器，它记录各个独立地图文件的位置关系，并不会改变原始地图文件本身。
- 3. **[如果项目里有几百张地图，还需要手动摆放吗？](https://bibigpt.co/search?q=%E5%A6%82%E6%9E%9C%E9%A1%B9%E7%9B%AE%E9%87%8C%E6%9C%89%E5%87%A0%E7%99%BE%E5%BC%A0%E5%9C%B0%E5%9B%BE%EF%BC%8C%E8%BF%98%E9%9C%80%E8%A6%81%E6%89%8B%E5%8A%A8%E6%91%86%E6%94%BE%E5%90%97%EF%BC%9F)**
  - 不一定。Tiled 的 World 功能支持 Pattern Match 高级用法，可以根据命名规则或模式自动计算坐标并摆放地图，适合超大规模地图项目。


### 术语解释
- **World（世界）**: Tiled 中用于组织多个地图文件的配置功能，可以显示和管理地图之间的空间关系。
- **子地图**: 被加入 World 文件中的独立地图文件，每张子地图仍然保持单独保存和编辑。
- **无缝世界**: 多张地图在视觉和逻辑上自然连接，玩家体验上像是在同一个连续的大地图中移动。
- **地图坐标**: World 文件中记录每张子地图摆放位置的数据，例如某张地图位于 `(0, 0)`，另一张地图位于 `(640, 0)`。
- **Pattern Match**: Tiled World 的高级功能，可根据规则自动匹配地图并计算它们在世界中的位置。
